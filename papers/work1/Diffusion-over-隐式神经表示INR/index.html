  • **φ(x)**：学习一个函数本身。该函数需要满足某些物理 / 几何 / 数学约束（方程）。

  • 约束：φ 及其导数必须满足某个条件

    C = 0

这里 φ 不是通过显式公式计算得到的，而是由一组“规则”所定义和约束的。

因此，INR 不是一个回归问题，而是一个：寻找合法函数的问题

（合法函数 = 满足所有约束条件的解）

在一组约束中：

寻找一个函数 φ满足m 条规则。不同规则的形式可能不同：

  • 有的只依赖 ∇φ(x)
  • 有的只依赖 ∇²φ(x)
  • 有的还需要外部已知量 a(x)

其中已知条件（PDE）通常由 PDE 给出，例如：

  • 源项（source term）
  • 边界条件（boundary conditions）

我们使用 Ω_m 表示第 m 条规则的生效区域，第 m 条约束并不要求在整个空间内都成立。

解释：

  • 总体结论：你有一张图像，你随机采样 (x, y)，对应的像素值就是 ground truth

那INR“没有 ground truth”？

因为：INR没有一个 “完整的连续函数” 的解析表达式，只有离散样本，网络需要从这些离散样本中 “重建” 连续函数

所以看起来像是 “没有 ground truth”，但实际上：ground truth 就是那些离散样本本身

而损失函数由积分 ==> 求和，在一堆随机采样电商，检查每条约束违反了多少，加起来就是loss

🌟 注意INR用偏微分方程PDE来监督网络，而PDE需要导数，NN可以对input求梯度【支持自动微分】 ==> 自动得到∇φ, ∇²φ ==> 这些提督可以参与loss

  • 因此PDE+NN可以直接训练，无需离散网络or显示有限差分

整体流程

随机采样x_i –> 前向传播 –> 自动求导 –> 代入所有C_m –> 计算违反程度 –> 反向传播 –> 更新参数

整个网络就是由多组这样的 “仿射 + 正弦” 层堆叠而成（也就是 Siren 网络）

  • 可以自然的表示高频细节
    • 普通激活（如 ReLU）对高频信号拟合差，而正弦函数本身是周期函数，能轻松表达快速变化的信号（比如流体、波的高频纹理）
  • 可以学习相同模式在不同输入坐标的移位不变性
    • 移位不变性 =“模式在空间中移动后，网络仍能识别 / 表示”（比如波在空间中传播，形状不变只位置变）
    • PDE 的解常具有平移不变的规律（如均匀介质中的波），正弦激活能天然适配这种规律，训练更高效
  • 可以同时拟合像素值及其导数
    • 正弦函数的任意阶导数都是光滑的（sin’=cos，cos’=-sin，无限可导），且自动微分能稳定计算其导数
    • 求解 PDE 需要同时用到函数值（φ）和各阶导数（∇φ、∇²φ），正弦激活的导数光滑且易算，不会出现 ReLU 导数不连续的问题

每一层都是用sin激活

场景

超分

图像本质上是一个 连续函数：I: [0,1] × [0,1] → [0,1]

这里的：x = 01 的坐标y = 01 的坐标

==> 归一化坐标，目的是把图像的像素位置映射到 [0,1] 的连续区间，让 INR 能以统一方式处理任意尺寸图像，同时契合 “图像是连续函数 I (x,y)” 的 INR 核心假设。

🌟 具体计算方法（以 32×32 图像为例）

  • 原始像素坐标：x∈[0,31]（列），y∈[0,31]（行）
  • 归一化公式：x_norm = x/(w-1)，y_norm = y/(h-1)（w=32，h=32）

结果：x_norm、y_norm 都落在 [0,1]，每个像素对应一个归一化坐标点，比如 (0,0) 是左上角，(1,1) 是右下角，(0.5,0.5) 是图像中心

(x, y) 是像素坐标，I (x,y) 是该位置的颜色

  • 低分辨率图像 = 对这个连续函数的稀疏、低质量采样

  • 高分辨率图像 = 对这个连续函数的密集、高质量采样

INR（比如 SIREN、MLP）的作用就是：学习这个连续函数 I (x,y)

一旦学到了连续函数，要生成 4K、8K、甚至任意分辨率都行 ==> 只要在更密集的坐标上查询 I (x,y) 即可

💡 为什么 INR 超分比传统方法强？

传统超分（ESPCN、EDSR、SRGAN）：

是 “离散→离散” 的映射，依赖卷积，感受野有限，因此难以生成高频细节，并且超分倍数受模型结构限制

INR 超分：

是 “连续函数拟合”，理论上支持任意倍数超分（4×、8×、64×…），能生成非常精细的高频纹理，并且不需要手工设计上采样滤波器

INR 超分不是 “猜像素”，而是 “恢复连续图像函数”。

通用INR超分方法（多图 非单图）

🔧 基于特征编码的通用 INR（如 LIIF, LTE）

为了解决“单图单权重”问题，现在的 INR 超分引入了 Encoder（编码器）。

  1. 先用一个通用的 CNN 或 Transformer（比如 EDSR 或 SwinIR）提取图像的特征图（Feature Map）

  2. INR 网络的输入不再仅仅是坐标 $(x, y)$，而是：**[坐标 + 该坐标对应的局部特征向量]**

  • 模型结构： $Output = f_\theta(Coordinate, Feature_{local})$。
  • 优势：
    • 一次训练，到处使用： 权重 $\theta$ 学习的是“如何利用特征进行插值和恢复细节”的通用规律，而不是某张图的像素
    • 任意倍率超分： 只要训练好了，你可以实现 1.372 倍、4.1 倍等非整数倍率的超分，这是传统 CNN 做不到的

❓ 为什么 WIRE 还要强调单图实验？

WIRE、SIREN 这类论文更偏向信号处理的底层基础。它们在单图上做实验，是为了在最严苛的条件下证明：

  1. 表达能力： 我的激活函数是不是比别人更能拟合高频细节
  2. 收敛速度： 拟合同一张图，我跑 500 次迭代的效果是不是比 SIREN 跑 5000 次还好
  3. 逆问题求解： 在医学影像（CT）或稀疏采样重建中，往往没有“大数据集”可供训练，必须依赖单样本的优化（Test-time Optimization），这时 WIRE 的局部性优势就是救命稻草
