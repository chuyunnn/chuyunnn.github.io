基本定义

Servlet 是Java EE规范中定义的请求处理基础组件，核心作用只有一个：

接收前端 HTTP 请求（如 GET/POST），调用后端业务逻辑，再把处理结果返回给前端。

它是前端和后端业务逻辑之间的 “中转站”，是 Web 项目的 “入口层”。

  • Servlet（LoginServlet）：只做 “请求接收” 和 “响应返回”，不关心 “怎么校验密码”；

  • 业务逻辑（UserService）：只做 “密码加密校验、用户状态判断”，不关心 “请求从哪来、响应怎么回”；

两者分工明确：Servlet 管 “和前端交互”，业务逻辑管 “实现具体功能”。

使用

一般是继承HttpServlet，然后覆盖doGet、doPost等方法

识别

1. @WebServlet

在自定义的servlet类上添加@WebServlet

urlPatterns = "/annotation"：表示当用户访问 http://localhost:8080/annotation 时，这个 Servlet 会处理请求 ==> 这一步相当于告诉容器：“这个类是 Servlet，负责 /annotation 路径的请求”

然后在springboot启动类上添加@SpringBootApplication

Spring Boot 默认不会扫描 @WebServlet 注解，必须加这个注解，Spring 才会去扫描并把你的 AnnotationServlet 注册到容器里 ==> 相当于告诉 Spring：“帮我把所有带 @WebServlet 的类都注册成 Servlet”

2. ServletRegistrationBean

当你的 Servlet 类没有任何注解（比如 @WebServlet）时，Spring Boot 无法自动扫描到它，这时就需要用 ServletRegistrationBean 手动把它注册到容器里
