解决让前端知道怎么调，如何快速测试的问题

RESTful web测试

测试接口能不能跑通

RESTful是一种流行的接口设计风格（GET、POST、PUT、DELETE）

测试方法：Knife4j的页面自带“发送请求”按钮，不需要postman或者网页，直接在生成的文档页面输入参数，点击发送就可以查看后台返回结果

Swagger

官网：https://swagger.io/

核心协议/规范，是API界的说明书标准

作用：自动扫描代码，比如springboot里的controller。根据代码中的注解生成一个包含所有**接口地址、参数要求、返回值格式**的文件 ==> 将项目中想要暴露的接口展示在页面上，然后就可以直接调用和测试

​ @Api 或 @ ApiOperation

格式：JSON 或 YAML格式

问题：

  • 原生界面Swagger-UI比较老

  • 布局不直观（接口多的时候）

  • 没有搜索功能

  • 请求参数为JSON的时候没办法格式化

  • 返回结构无法折叠

Knife4j

官网：https://doc.xiaominfo.com/

码云地址：https://gitee.com/xiaoym/knife4j

示例地址：https://gitee.com/xiaoym/swagger-bootstrap-ui-demo

针对java生态的Swagger的增强版

  • 使用Swagger的逻辑来解析接口，重新做了一套UI界面

优点：

  • 在API文档中使用Markdown语法，可以使文档更具可读性和易于维护

  • 将API文档导出为离线的HTML、PDF或Markdown文件，方便分享

  • 支持在不同的环境（比如开发、测试、生产）中使用不同的 API 文档配置

  • 支持动态参数，允许用户在运行时修改 API请求的参数，提高测试的灵活性

  • 界面清晰：树形菜单+接口详情

  • 支持接口全局搜索、参数缓存、方便的Debug调试界面

项目整合

因为Knife4j遵循Swagger 的使用方式，所以可以无缝切换

  1. 添加依赖 pom.xml
  1. 创建java配置类，比如Knife4jConfig.java，使用@EnableKnife4j注解启用Knife4j

或者在application.yml文件中设置属性（结果相同）

  1. 在测试类中添加Knife4j的接口
  1. 运行项目，输入http://localhost:8080/doc.html 就可以看到API文档了
