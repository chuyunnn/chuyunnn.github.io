基本定义

Listener是web中的事件监听器，能监听web应用中特定事件的发生，并在事件触发时自动执行预设的逻辑，并且不侵入业务代码

比如：

  • 应用启动的的时候加载全局配置、初始化缓存
  • 用户第一次访问的时候初始化session
  • 请求结束的时候记录访问日志

特点：

  • 不会主动执行，只有当对应事件发生时才会被容器调用
  • 可以监听整个应用、用户会话或单个请求的生命周期，适合做全局监控和初始化

常见的Listener类型

Java Web 规范定义了多种标准监听器，按监听对象可分为四类：

类型 作用 典型场景
ServletContextListener 监听 ServletContext（应用全局上下文）的创建与销毁 应用启动时加载配置、初始化连接池；应用关闭时释放资源
HttpSessionListener 监听 HttpSession（用户会话）的创建与销毁 **统计在线用户数**；用户登录 / 退出时更新状态
ServletRequestListener 监听 ServletRequest（请求）的创建与销毁 **记录请求耗时**；统一处理请求日志
AttributeListener 监听 ServletContext/Session/Request 中属性的增删改 监控全局属性变化；实现属性缓存刷新

使用

使用@WebListener标记是一个web监听器

如果使用了@WebListener注解，要在springboot启动类上添加 @ServletComponentScan，让 Spring 自动扫描并注册监听器

如果监听器类没有注解，可以用ServletListenerRegistrationBean手动注册
