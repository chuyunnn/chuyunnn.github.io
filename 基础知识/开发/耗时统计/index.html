查看http请求耗时

统计整个请求中每个阶段的耗时：直接借助 curl 来实现

参数解析说明

  • -o /dev/null 把返回值丢掉，不用输出
  • -s 静默输出，不输出进度条
  • -w 指定格式打印信息，其中包含一些特定的参数：
    • time_namelookup：从开始到域名解析完成时的耗时
    • time_connect：从开始到 TCP 连接建立完成的耗时
    • time_appconnect：从开始到 TLS 连接建立完成的耗时
    • time_redirect：多次重定向（如果有）的耗时
    • time_pretransfer：从开始到准备发送请求消息前的耗时
    • time_starttransfer：从开始到服务器准备返回第一个字节时的耗时
    • time_total：整个 HTTP 请求操作耗时

stopwatch

【方法内部多个子模块 / 代码块的耗时】

使用spring/hutool的 stopwatch 统计耗时

使用步骤：

  1. 初始化StopWatch实例

  2. 启动计时（可直接启动总计时，或给分段计时命名） stopWatch.start();

  3. 执行待统计的代码模块 stopWatch.stop();

  4. 停止当前分段计时（分段统计时用），启动下一个分段计时

  5. 所有模块执行完成后，打印 / 获取各分段耗时、总耗时

    • 用prettyPrint()格式化打印

    • 用getTotalTimeMillis()获取总耗时

核心禁忌：不要重复启动未停止的计时器、不要停止未启动的计时器

Filter记录请求耗时

【请求】

基于 Servlet Filter 的全局拦截方案，在请求进入和离开时分别记录时间，计算差值得到总耗时

  1. 请求进入时：在 doFilter 方法开头记录当前时间戳 startTime = System.currentTimeMillis()
  2. 请求结束时：在 filterChain.doFilter(request, response) 执行完成后，计算耗时 costTime = System.currentTimeMillis() - startTime
  3. 日志记录：调用 buildRequestLog 方法，将请求方法、URL、用户 ID、耗时等信息拼接成日志格式并输出

自定义注解，结合AOP

【单个业务方法】

基于 Spring AOP + 自定义注解 的方法级统计方案，通过切面拦截被注解标记的方法，计算其执行耗时

注解标记：在需要统计的方法上添加 @MdcDot 注解

AOP 拦截：@Around 切面在方法执行前后分别记录时间戳
