websocket

Created2026-01-30|Updated2026-01-30|基础知识开发
|Post Views:

定义

  1. 基于 TCP 连接的全双工通信协议 ==> 客户端和服务器可以同时发送和接收数据

  2. 应用层协议，用于弥补 HTTP 协议在**持久通信能力**上的不足

  • 客户端和服务器仅需一次握手，两者之间就直接可以创建持久性的连接，并进行双向数据传输

和HTTP的区别

websocket http
双向实时通信协议（不存在只能有一方主动） 单向通信协议，并且通信只能由客户端发起，服务器无法主动通知客户端
使用 ws:// 或 wss://作为协议前缀（加密or不加密） 使用 http:// 或 https:// 作为协议前缀
支持扩展。用户可以扩展协议，实现部分自定义的子协议，如支持压缩、加密等 不支持
通信数据格式比较轻量，用于协议控制的数据包头部相对较小，网络开销小 每次都要携带完整的头部，网络开销较大

短轮询、长轮询

1.短轮询（Short Polling）

  • 原理：客户端每隔固定时间（如 5 秒）发起一次 HTTP 请求，询问服务器是否有新数据。服务器收到请求后立即响应
  • 优点：实现简单，兼容性好，直接用常规 HTTP 请求即可
  • 缺点：
    • 实时性一般：消息可能在两次轮询间到达，用户需等到下次请求才知晓
    • 资源浪费大：反复建立/关闭连接，且大多数请求收到的都是“无新消息”，极大增加服务器和网络压力

2.长轮询（Long Polling）

  • 原理：客户端发起请求后，若服务器暂时无新数据，则会保持连接，直到有新数据或超时才响应。客户端收到响应后立即发起下一次请求，实现“伪实时”
  • 优点：
    • 实时性较好：一旦有新数据可立即推送，无需等待下次定时请求
    • 空响应减少：减少了无效的空响应，提升了效率
  • 缺点：
    • 服务器资源占用高：需长时间维护大量连接，消耗服务器线程/连接数
    • 资源浪费大：每次响应后仍需重新建立连接，且依然基于 HTTP 单向请求-响应机制
