后端是如何返回JSON/XML的

返回文本数据

  1. controller上添加@RestController注解：直接返回json
  2. 如果controller注解是@Controller，可以添加@ResponseBody注解实现返回json
  3. 通过HttpServletResponse方法将返回数据手动写入到输出流中。比如前面全局异常处理同时支持返回页面+json数据两种方式，在全局异常处理中可以进行分条件输出

返回网页

网页实际上也是一种文本，返回网页更准确的说法是返回渲染视图。

接口实现：Controlleer上的注解是@Controller，没有@RestController，返回一个网页模板的相对路径（string）

返回XML

可以实现一个rest服务

  • @RestController：返回的不是视图
  • @PostMapping注解中的 consumes 和 produces参数，指定了”application/xml”，表示接收和返回的都是xml文档
  • @RequestBody是用来告诉 Spring MVC：把整个 HTTP 请求体（Body）的内容，转换成你指定的 Java 对象（这里是 XmlBean）。XML 参数是放在请求体里的，如果不加这个注解，Spring MVC 根本不会去解析请求体里的 XML 内容，自然就无法给 XmlBean 赋值 ==> 这里是接收参数

@RestController：管出，也就是返回响应

@RequestBody：管进，也就是接受参数
